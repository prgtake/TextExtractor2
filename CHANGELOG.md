# Changelog

すべての変更はこのファイルに記録されます。

## [1.1.0] - 2026-05-18

### Added
- **逐次リサーチ（Sequential Staging）アーキテクチャ**: Gemini API のツール同時使用制限を回避するため、WEB検索・RAG検索・SQLiteツールを個別に実行し、最後に統合するパイプラインを実装。
- **SQLite スキーマ自動解析**: 接続されたデータベースの構造（テーブル名・カラム名）を自動取得し AI に提示することで、誤った SQL の発行を防止。
- **マルチターン・エージェント機能**: SQLite ツール実行時に AI が複数回の対話を行えるループ処理を実装し、より複雑なデータ取得に対応。

### Improved
- **データ取得精度の向上**: AI に対し、氏名のスペースの有無などの表記揺れを考慮した柔軟な検索（LIKE 等）を行うよう指示を強化。
- **堅牢な JSON 生成**: ツール使用時は JSON Mode を動的に切り替えるロジックを導入し、パースエラーの発生率を大幅に低減。

### Fixed
- **API 制限の回避**: Gemini API において「WEB検索/RAG」と「JSON Mode」が併用できない問題、および「WEB検索」と「Function Calling」が併用できない仕様制限を、逐次処理パイプラインにより解決。
- **データ型エラーの修正**: ツール実行時の有効カウント処理において発生していた `TypeError (int and str)` を修正。

## [1.0.0] - 2026-05-10

### Added (Initial Release)
- **サブフォルダ単位の統合解析**: フォルダ内の複数ファイルを 1 つの文脈として AI に読み取らせるコアロジックを実装。
- **AI 指示文相談チャット**: Gemini が「指示文アドバイザー」として、最適な抽出項目やルールを提案する機能を搭載。
- **SQLite データベース自動生成**: 抽出結果を `.db` 形式で永続化し、中断・再開が容易な「処理済みスキップ」機能を実装。
- **SQLite ツール連携**: プロンプト内から外部 DB を参照・更新できる強力なエージェント機能を搭載。
- **多彩な抽出エンジン**: PDF, 画像, Office 文書に加え、一太郎, RTF, ODT, メール形式 (.eml/.msg) に対応。
- **洗練された UI**: 横幅を統一したボタン、コンパクトなウィンドウ、リアルタイム実行ログ表示。
- **堅牢なリトライロジック**: API の 429/503 エラーに対し、最大 5 回の指数バックオフ付き再試行を実装。
- **自動ビルド環境**: Python 3.13 に対応した `build_textextractor2.bat` および依存関係管理の提供。
