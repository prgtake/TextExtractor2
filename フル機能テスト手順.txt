TEST手順

? textextractor2 の全機能（マルチ形式抽出、DB連携、RAG検索、WEB検索）を一括でテストする手順は以下の通りです。

  1. 事前準備（RAG資料のアップロード）
  RAG機能をテストするために、以下のファイルを Google AI Studio 等の管理画面から「File Search
  Store」にアップロードしてください。
   - TEST（RAG用資料）/company_policy_2026.txt
   - TEST（RAG用資料）/technical_glossary.txt
  ※アップロード後、作成されたストアの名前（ID）を控えておいてください。

  2. ツールの設定
  textextractor2.py を起動し、メイン画面で以下の設定を行います。

   1. 処理対象フォルダ: TEST（フル機能） フォルダを選択。
   2. 指示ファイル(txt): TEST（フル機能）/prompt_all_in_one.txt を選択。
   3. 実行オプション:
      - 「WEB検索を利用」: チェックを入れる。
      - 「RAG(File Search)を利用」: チェックを入れる。
      - 「SQLiteツールを利用」: チェックを入れる。
   4. 使用するRAG: 手順1で作成したストアをドロップダウンから選択。

  3. テスト実行と結果の確認
  「指示文を実行」 ボタンをクリックします。完了後、以下の項目が正しく抽出・生成されているか確認してください。

  ┌──────────────────────────────────────────────
  │ 項目         │ 検証内容 (期待される動作)                                                  │
  ├──────────────────────────────────────────────
  │ 氏名         │ WordやPDFから「田中太郎」等が正しく抽出されているか。                      │
  │ 役割・状態   │ master_data.db を参照し、田中なら「リーダー/現役」となっているか。         │
  │ 合計点数     │ Excel内の点数をAIが正しく合計できているか。                                │
  │ 表彰候補     │ (RAG) 規定資料に基づき、田中（リーダー）が「MVP」と判定されるか。          │
  │ 評価コメント │ (RAG) 資料内の「山登り」を「プロジェクト管理」と解釈して記載されているか。 │
  │ 備考         │ (WEB) 田中太郎という名前の一般的さや由来がネット検索から引用されているか。 │
  └──────────────────────────────────────────────
  この手順で、textextractor2
  が持つすべてのインテリジェントな機能を一つのワークフローで網羅的にテストすることが可能です。
